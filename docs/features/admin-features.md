# 관리자 기능 명세

이 문서는 관리자와 운영자가 사용하는 관리 화면, 운영 도구, 로그/진단 기능을 정리합니다.

## 1. 관리자 대시보드

- 목적: 운영 상태와 자주 쓰는 관리 기능의 진입점을 제공합니다.
- 대상 사용자: 관리자
- 진입 경로: `/admin`
- 핵심 동작:
  - 주요 기능 바로가기
  - 운영 요약 정보 표시
  - 관리자 전용 작업 허브 역할

## 2. 사용자 관리

- 목적: 사용자 계정과 역할을 관리합니다.
- 대상 사용자: 관리자
- 진입 경로: `/admin/users`
- 핵심 동작:
  - 학생/졸업생/교사/방송부/관리자 그룹별 조회
  - 계정 상태 관리
  - 기존 계정 역할 변경
  - 학생/방송부/졸업생 계정 기수 변경
  - 사용자 삭제
  - 사용자 내보내기
- 데이터 저장: `User`
- 회귀 시 주의할 점:
  - 역할 변경은 방송부/관리자 접근성에 직접 영향
  - 관리자 전환 시 `gisu`는 제거되고 `studentId`는 유지
  - 졸업생은 학생 메타를 유지하지만 학생 전용 핵심 정보 접근은 제한
  - 자기 자신 관리자 해제와 마지막 관리자 해제/삭제는 차단되어야 함

## 3. 공지 및 카테고리 관리

### 공지 관리

- 경로: `/admin/notices`
- 목적: 공지 생성, 수정, 만료 관리
- 데이터 저장: `Notice`

### 카테고리 관리

- 경로: `/admin/categories`
- 목적: 공지 카테고리와 표시 체계 유지
- 데이터 저장: `NoticeCategory`

회귀 시 주의할 점:

- 카테고리 데이터가 비어도 공지 작성 페이지가 서버 오류로 죽지 않아야 함

## 4. 토큰 발급 및 메일 발송

- 목적: 역할별 회원가입 토큰과 메일 발송을 관리합니다.
- 대상 사용자: 관리자
- 진입 경로: `/admin/tokens`
- 핵심 동작:
  - 배치 발급
  - 역할별 수동 메일 발송
  - 최근 발송 로그 조회
  - 발송 성공/실패, Brevo 메시지 ID 확인
- 데이터 저장:
  - `InviteToken`
  - `TokenBatch`
  - `TokenDistributionLog`
- 관련 설정:
  - 토큰 배부 포털 활성화
  - 포털 비밀번호
  - 추가 안내 문구
  - 오늘 발송량 집계

## 5. 공용 설정과 운영 설정

- 목적: 서비스 전반의 운영 설정을 DB 기반으로 관리합니다.
- 대상 사용자: 관리자
- 진입 경로: `/admin/settings`
- 핵심 동작:
  - Google Analytics 관리
  - 토큰 배부 포털 상태/비밀번호/안내 문구 관리
  - 백업 다운로드/복원
  - 운영 관련 설정 값 저장
- 데이터 저장:
  - `SystemSetting`
- 회귀 시 주의할 점:
  - 설정 저장 실패가 페이지 전체 크래시로 이어지면 안 됨
  - 컨테이너 안에서도 백업 경로와 임시 파일 경로가 writable이어야 함

## 6. 사이트, 기상곡, 알림 운영

### 사이트 관리

- 경로: `/admin/sites`
- 목적: 교내 사이트 목록 관리
- 데이터 저장: `RelatedSite`

### 기상곡 운영

- 경로: `/admin/songs`, `/music`
- 목적:
  - 전체 운영 관점에서 기상곡 요청 관리
  - 방송부/관리자의 검수 및 상태 변경
- 데이터 저장: `SongRequest`, `SongRule`
- 회귀 시 주의할 점:
  - 요일별 신청 제한은 KST 기준 유지
  - 승인/반려 흐름과 알림을 조용히 없애지 않기

### 알림 운영

- 경로: `/admin/notifications`
- 목적: 시스템/운영 알림 관리

## 7. 로그, 리포트, 진단

### 로그

- 경로: `/admin/logs`
- 목적: 사용자/시스템 로그 탐색
- 데이터 저장: `SystemLog`, `AuditLog`

### 오류 리포트

- 경로: `/admin/reports`
- 목적: 사용자가 올린 오류 신고 확인 및 처리
- 데이터 저장: `ErrorReport`

### 운영 진단

- 경로: `/admin/test`
- 목적: 운영 준비 상태와 핵심 진단 항목을 한 번에 확인
- 대표 점검 항목:
  - DB 연결
  - NEIS 연동
  - 로그 시스템
  - DB 쓰기
  - 기상곡 흐름
  - 현재 앱 버전
  - 백업 디렉터리 쓰기 가능 여부
  - 최근 백업 시각
  - 디스크 여유
  - `DATABASE_URL` 경로 적합성
- 회귀 시 주의할 점:
  - 운영 직전 최종 확인 기준이므로 PASS/FAIL 기준을 함부로 약화하지 않기

## 8. 운영 안전장치와 관련된 관리자 기능

- 토큰 배부 포털은 관리자 설정에서만 켜고 끌 수 있음
- 백업/복원은 관리자 설정과 서버 배포 구조에 함께 의존함
- Brevo 발송량은 앱 로그 기준 일일 집계로 관리
- 서비스 버전은 푸터와 GitHub Release에 함께 노출됨

## 9. 관리자 기능 변경 시 공통 체크포인트

- 권한 체크가 `ADMIN` 또는 `BROADCAST` 조건과 맞는지
- 저장 실패 시 사용자에게 안전한 오류를 보여주는지
- 테스트/운영 도메인 값이 섞이지 않는지
- 운영 문서와 실제 동작이 여전히 일치하는지
