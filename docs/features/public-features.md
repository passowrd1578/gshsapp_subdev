# 공개 기능 명세

이 문서는 비로그인 상태에서도 접근 가능한 화면과 공개 정보 기능을 정리합니다.

## 1. 홈

- 목적: 학교 생활에서 자주 보는 핵심 정보를 첫 화면에서 빠르게 보여줍니다.
- 대상 사용자: 모든 사용자
- 진입 경로: `/`
- 핵심 동작:
  - 오늘의 급식 요약
  - 일정 요약
  - 공지 배너 또는 최근 공지
  - 빠른 이동 링크
  - 로그인 상태라면 개인 일정이 없어도 오늘의 공용 일정 요약을 함께 표시
- 데이터 출처:
  - 급식: NEIS Open API 캐시 데이터
  - 일정: 내부 일정 + Google Calendar iCal + NEIS 학사일정
  - 공지: 내부 Notice 데이터
- 회귀 시 주의할 점:
  - 홈은 공개 렌더링 성능에 민감함
  - 로그인 여부 때문에 공개 렌더 경로가 강제 동적이 되지 않도록 주의
  - 일정 없음 문구는 `오늘은 일정이 없습니다.` 기준 유지

## 2. 공지사항

- 목적: 학교 공지와 운영 공지를 공개적으로 제공합니다.
- 대상 사용자: 모든 사용자
- 진입 경로: `/notices`, `/notices/[id]`
- 핵심 동작:
  - 목록 조회
  - 상세 조회
  - 카테고리/작성자/작성일 정보 표시
- 데이터 출처: 내부 Notice 데이터
- 갱신 기준: 공개 캐시 기준 약 5분 이내 반영
- 관련 관리자 기능: `/admin/notices`, `/admin/categories`
- 회귀 시 주의할 점:
  - 관리자 작성 공지와 교사/일반 공지의 표기 차이를 유지
  - 카테고리 데이터가 비어 있어도 페이지가 크래시하면 안 됨

## 3. 급식

- 목적: 오늘과 예정 급식을 빠르게 확인하게 합니다.
- 대상 사용자: 모든 사용자
- 진입 경로: `/meals`
- 핵심 동작:
  - 날짜별 급식 조회
  - 홈/급식 페이지에서 요약 정보 제공
  - 조회 기록을 통계에 반영
- 데이터 출처: NEIS Open API
- 갱신 기준: 캐시 약 1시간
- 관련 관리자 기능: 없음
- 회귀 시 주의할 점:
  - 급식 API 실패 시에도 페이지 전체가 죽지 않아야 함
  - 조회 로깅은 UI를 막지 않는 비차단 방식 유지

## 4. 시간표

- 목적: 학생들이 날짜 기준 시간표를 확인합니다.
- 대상 사용자: 로그인 사용자, 졸업생 제외
- 진입 경로: `/timetable`
- 핵심 동작:
  - 날짜 기준 시간표 조회
  - 사용자 상태에 따라 관련 정보 보강 가능
- 데이터 출처: 학교 시간표 데이터 및 앱 내부 가공 데이터
- 회귀 시 주의할 점:
  - 날짜 처리와 로그인 접근 제어를 함께 유지
  - 졸업생은 로그인해도 접근할 수 없도록 유지
  - 홈 개인화 시간표와 공개 시간표 페이지를 혼동하지 않기

## 5. 학사일정

- 목적: 시험, 행사, 휴일 등 학교 일정을 달력 기반으로 제공합니다.
- 대상 사용자: 모든 사용자
- 진입 경로: `/calendar`
- 핵심 동작:
  - 월 단위 달력 보기
  - 일정 목록 및 상세 요약
  - 홈 D-day 요약과 연결
- 데이터 출처:
  - 내부 Schedule 데이터
  - Google Calendar iCal
  - NEIS 학사일정
- 갱신 기준:
  - 앱 내부 캐시 약 5분
  - 일부 NEIS 소스 캐시 약 24시간
- 관련 관리자 기능: 일정 생성/수정 흐름이 있는 경우 관리자 또는 내부 데이터 관리
- 회귀 시 주의할 점:
  - 초기 날짜가 서버/클라이언트에서 어긋나면 하이드레이션 오류가 날 수 있음

## 6. 기상곡

- 목적: 학생이 기상곡을 신청하고 공개적으로 목록을 확인합니다.
- 대상 사용자: 로그인 사용자, 졸업생 제외
- 진입 경로: `/songs`
- 핵심 동작:
  - 곡 목록 조회
  - 유튜브 썸네일과 제목 표시
  - 썸네일/제목 클릭 시 원본 YouTube 새 탭 열기
  - 로그인 사용자는 신청 폼 접근
- 데이터 출처: 내부 SongRequest 데이터 + YouTube URL 메타데이터
- 관련 관리자 기능: `/music`, `/admin/songs`
- 회귀 시 주의할 점:
  - 신청 제한은 KST 기준 요일 규칙을 따름
  - 방송부 스튜디오의 요일별 신청 학년 설정이 신청 페이지 안내와 동일하게 보여야 함
  - 졸업생은 로그인해도 접근할 수 없도록 유지
  - 관리자/방송부/본인에게만 추가 메타데이터가 보여야 함

## 7. 링크모음

- 목적: 자주 쓰는 학습/학교/외부 링크를 모아 제공합니다.
- 대상 사용자: 로그인 사용자, 졸업생 제외
- 진입 경로: `/links`
- 핵심 동작:
  - 링크 목록 조회
  - 교사/관리자는 링크 추가/수정/삭제 가능
- 데이터 출처: 내부 LinkItem 데이터
- 관련 관리자 기능: 링크 편집 액션
- 회귀 시 주의할 점:
  - 현재 명칭은 `바로가기 모음`이 아니라 `링크모음`
  - 졸업생은 로그인해도 접근할 수 없도록 유지

## 8. 교내 사이트

- 목적: 학교 내부 또는 커뮤니티 관련 사이트를 모아 보여줍니다.
- 대상 사용자: 로그인 사용자, 졸업생 제외
- 진입 경로: `/sites`
- 핵심 동작:
  - 교내 사이트 목록 조회
  - 카테고리별 묶음
- 데이터 출처: 내부 RelatedSite 데이터
- 관련 관리자 기능: `/admin/sites`
- 회귀 시 주의할 점:
  - 현재는 로그인하지 않으면 접근할 수 없도록 보호됨
  - 졸업생은 로그인해도 접근할 수 없도록 유지

## 9. 도구

- 목적: 학내에서 간단히 쓸 수 있는 유틸리티를 제공합니다.
- 대상 사용자: 모든 사용자
- 진입 경로: `/utils`
- 포함 기능:
  - 바이트 계산기
  - 랜덤 숫자
  - 자리 뽑기
  - 타이머
  - 스톱워치
- 회귀 시 주의할 점:
  - 각 도구는 `/utils`로 돌아가는 공통 뒤로가기 UI를 유지
  - 자리 뽑기의 학년/반 표시는 숫자 좌석 라벨 규칙과 같이 유지

## 10. 도움말, 개인정보처리방침, 통계

### 도움말

- 경로: `/help`
- 목적: 사용법, 문의/제안, 데이터 출처와 갱신 기준 안내
- 회귀 시 주의할 점:
  - 오류 신고는 `/report`와 분리
  - 문의/제안 섹션과 데이터 출처 섹션을 유지

### 개인정보처리방침

- 경로: `/privacy`
- 목적: 실제 수집 정보와 처리 기준 안내

### 통계

- 경로: `/stats`
- 목적: 서비스 이용 흐름과 주요 사용량 지표 시각화
- 회귀 시 주의할 점:
  - 문구 깨짐이나 인코딩 오류가 재발하지 않도록 주의
