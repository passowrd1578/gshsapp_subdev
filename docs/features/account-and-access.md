# 계정 및 접근 기능 명세

이 문서는 로그인, 회원가입, 토큰 발급, 개인화, 알림, 오류 신고처럼 계정과 접근 제어에 관련된 기능을 정리합니다.

## 1. 로그인

- 목적: 사용자 역할과 개인화 기능 접근을 가능하게 합니다.
- 대상 사용자: 학생, 졸업생, 교사, 방송부, 관리자
- 진입 경로: `/login`
- 핵심 동작:
  - 아이디/비밀번호 로그인
  - 이미 로그인된 사용자의 적절한 우회 처리
- 관련 관리자 기능: 사용자 계정 관리, 토큰 발급
- 회귀 시 주의할 점:
  - 테스트/운영 도메인 리다이렉트가 섞이면 안 됨
  - 비로그인 상태에서 상단 테마 토글은 보여도, 계정 메뉴는 로그인 상태에 맞게 보여야 함

## 2. 토큰 기반 회원가입

- 목적: 초대 토큰을 받은 사용자만 계정을 생성하게 합니다.
- 대상 사용자: 학생, 졸업생, 교사, 방송부, 관리자
- 진입 경로: `/signup?token=...`
- 핵심 동작:
  - 토큰 검증
  - 역할/기수 검증
  - 비밀번호 설정과 계정 생성
  - 토큰 사용 처리
- 데이터 저장:
  - `InviteToken`
  - `User`
- 관련 관리자 기능: `/admin/tokens`
- 회귀 시 주의할 점:
  - 토큰 발급 후 가입 단계는 토큰을 직접 배부할 때와 동일해야 함
  - 토큰 만료 규칙과 사용 처리 규칙을 바꾸지 않음

## 3. 토큰 배부 포털

- 목적: 학생 대상 토큰 메일 발송을 관리자가 켰을 때만 공개적으로 수행합니다.
- 대상 사용자: 학생용 토큰을 요청하는 사용자, 관리자
- 진입 경로: `/signup/request`
- 핵심 동작:
  - 포털 켜기/끄기
  - 관리자 설정 비밀번호 확인
  - 이름, 학번, 이메일 입력
  - Brevo API를 통한 토큰 및 가입 링크 발송
  - 기존 미사용 토큰 재사용
- 데이터 저장:
  - `InviteToken`
  - `TokenDistributionLog`
  - 관련 `SystemSetting`
- 관련 관리자 기능:
  - `/admin/settings`의 포털 상태/비밀번호/안내 문구/오늘 발송량
  - `/admin/tokens`의 수동 발송 및 로그
- 회귀 시 주의할 점:
  - 공개 포털은 학생 전용
  - 일일 발송량 제한은 현재 앱 기록 기준 250건
  - 같은 IP 제한이 아니라 클라이언트 세션 기반 쿨다운을 사용

## 4. 수동 토큰 발송

- 목적: 관리자가 학생, 교사, 방송부, 관리자용 토큰을 직접 메일로 보낼 수 있게 합니다.
- 대상 사용자: 관리자
- 진입 경로: `/admin/tokens`
- 핵심 동작:
  - 이메일 입력
  - 역할 선택
  - 학생 역할일 경우 기수 지정
  - 발송 성공/실패 로그 기록
- 데이터 저장:
  - `InviteToken`
  - `TokenDistributionLog`
- 회귀 시 주의할 점:
  - 공개 포털과 수동 발송의 역할 범위가 다름
  - Brevo 실패 시 새로 만든 토큰은 롤백 삭제되어야 함

## 5. 내 정보와 개인화

- 목적: 로그인한 사용자의 계정 정보와 개인 일정을 제공합니다.
- 대상 사용자: 로그인 사용자
- 진입 경로: `/me`
- 핵심 동작:
  - 사용자 기본 정보 표시
  - 개인 일정 관리
  - 모바일 전용 로그아웃 진입점 제공
- 데이터 저장:
  - `User`
  - `PersonalEvent`
- 회귀 시 주의할 점:
  - 개인 일정이 없을 때 홈 문구는 `오늘은 일정이 없습니다.` 기준 유지

## 6. 역할과 접근 변경

- 목적: 가입 후에도 관리자 판단에 따라 계정 역할과 기수를 안전하게 조정합니다.
- 대상 사용자: 관리자, 로그인 사용자
- 진입 경로:
  - 사용자 본인: 로그인 후 적용 결과 사용
  - 관리자: `/admin/users`
- 핵심 동작:
  - 기존 계정을 `STUDENT`, `GRADUATE`, `TEACHER`, `BROADCAST`, `ADMIN`으로 변경
  - 학생, 방송부, 졸업생 계정의 기수 직접 변경
  - 자기 자신 관리자 해제 차단
  - 마지막 관리자 계정 해제/삭제 차단
- 데이터 저장:
  - `User`
  - `SystemLog`
- 관련 관리자 기능: `/admin/users`
- 회귀 시 주의할 점:
  - `GRADUATE`는 로그인은 가능하지만 학생 전용 핵심 정보 접근은 제한
  - 학생 전환 시 `studentId` 4자리 형식과 `gisu` 자동 계산 규칙을 유지
  - 관리자 권한 변경 직후 서버가 DB 기준 최신 역할을 다시 확인해야 함

## 7. 알림

- 목적: 공지, 시스템, 기상곡 상태 등 사용자에게 필요한 알림을 제공합니다.
- 대상 사용자: 로그인 사용자
- 진입 경로: `/notifications`
- 핵심 동작:
  - 알림 목록 조회
  - 읽음 처리
  - 읽은 알림 모두 삭제
  - 헤더/메뉴 배지 갱신
- 데이터 저장: `Notification`
- 관련 관리자 기능: `/admin/notifications`
- 회귀 시 주의할 점:
  - 알림 배지는 공개 셸 사용자 요약과 연동됨
  - 읽은 알림 삭제는 읽지 않은 알림까지 지우면 안 됨

## 8. 오류 신고와 문의 분리

### 오류 신고

- 경로: `/report`
- 목적: 실제 버그나 장애를 신고
- 데이터 저장: `ErrorReport`

### 일반 문의/기능 제안

- 경로: `/help#contact`
- 목적: 문의/제안 안내와 연락 방법 제공
- 현재 기본 안내 채널: `admin@gshs.app`

회귀 시 주의할 점:

- `/report`는 오류 신고 전용 유지
- 일반 문의/제안은 도움말에서 처리

## 8. 권한 모델

현재 주요 역할:

- `STUDENT`
- `GRADUATE`
- `TEACHER`
- `BROADCAST`
- `ADMIN`

역할별 대표 접근:

- 학생: 개인 기능, 기상곡 신청
- 졸업생: 로그인과 기본 계정 기능은 가능하지만 학생 전용 핵심 정보는 제한
- 교사: 링크모음 편집, 공지 작성
- 방송부: 방송부 스튜디오, 기상곡 검수
- 관리자: 운영 도구 전체

권한 관련 회귀 시 특히 확인할 경로:

- `/login`
- `/signup`
- `/signup/request`
- `/me`
- `/notifications`
- `/admin`
- `/music`
