## 1. 제품 목적

GSHS.app은 경남과학고등학교 구성원이 자주 확인하는 학교 정보를 한곳에서 빠르게 조회하고, 필요한 운영 기능까지 함께 수행할 수 있도록 만든 통합 웹 서비스입니다.

핵심 목표:

- 공지, 급식, 시간표, 학사일정 같은 반복 조회 정보를 빠르게 제공
- 기상곡 신청, 링크모음, 교내 사이트처럼 학생 생활에 자주 쓰이는 기능을 한곳에 모으기
- 관리자와 방송부가 별도 도구 없이 운영 작업을 수행할 수 있게 하기
- 테스트/운영 배포와 백업, 복원, 릴리스 추적까지 일관된 운영 체계를 유지하기

## 2. 사용자 역할

### 비로그인 사용자

- 공개 페이지 조회 가능
- 공지, 급식, 기상곡, 시간표, 학사일정, 링크모음, 통계, 도움말 사용 가능
- 회원가입은 토큰 기반이며, 토큰 배부 포털 또는 관리자 발급을 통해 시작

### 학생

- 비로그인 기능 + 로그인 기반 개인화 기능 사용
- 내 정보, 개인 일정, 알림, 오류 신고 사용
- 기상곡 신청 가능
- 시간표, 링크모음, 교내 사이트 접근 가능

### 졸업생

- 로그인과 기본 계정 기능 사용 가능
- 내 정보, 알림, 오류 신고 같은 기본 계정 기능은 유지
- 학생 전용 핵심 정보인 기상곡, 시간표, 링크모음, 교내 사이트 접근은 제한

### 교사

- 학생 기능 + 링크모음 관리, 공지 작성 지원

### 방송부

- 학생 기능 + 기상곡 검수 및 방송부 스튜디오 접근

### 관리자

- 전 기능 접근 가능
- 사용자, 공지, 카테고리, 토큰, 설정, 사이트, 로그, 리포트, 운영 진단 관리

## 3. 핵심 사용자 흐름

### 공개 정보 조회

1. 홈에서 오늘의 급식, 일정, 공지, 빠른 이동 확인
2. 공지사항, 급식, 학사일정, 도구, 통계 화면으로 이동
3. 필요한 경우 로그인 또는 오류 신고/문의로 연결

### 토큰 기반 회원가입

1. 관리자가 토큰을 직접 발급하거나 토큰 배부 포털을 켬
2. 사용자는 메일로 토큰과 회원가입 링크를 받음
3. `/signup?token=...`에서 기존 규칙대로 가입을 완료
4. 이후 로그인해 개인 기능을 사용

### 기상곡 신청과 운영

1. 학생이 `/songs`에서 기상곡을 신청
2. 방송부 또는 관리자가 `/music`에서 검수
3. 관리자/방송부/본인에게는 더 많은 메타데이터가 노출됨
4. 상태 변경에 따라 알림과 운영 흐름이 이어짐

### 운영 관리

1. 관리자가 `/admin`에서 운영 도구 진입
2. 사용자/공지/토큰/설정/로그/리포트/백업 상태 확인
3. 기존 계정의 역할 변경, 기수 변경, 사용자 삭제 수행
3. `/admin/test`로 운영 준비 상태 점검
4. 배포는 GitHub Actions와 self-hosted runner 기준으로 진행

## 5. 현재 제품에서 중요하게 유지하는 규칙

- 테스트와 운영 도메인 동작을 섞지 않음
- 운영 배포는 `sha-<commit>` 기준으로만 승격
- GitHub Release는 `vX.Y.Z` semver 기준으로 관리
- Google Analytics는 관리자 설정에서 제어
- SQLite는 영속 볼륨 위에서만 운영
- 관리자/방송부/교사 권한 차이를 조용히 변경하지 않음
